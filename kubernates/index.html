Kubernetes

  • local deploy: minikube
  • cloud deploy: kops
  • management cli: kubectl

MiniKube

minikube start
minikube stop

Kops

  1. create domain and delegate to Route 53: k.devops.it
  2. create an S3 bucket to store the state of kops (eg: kops-state-dev.devops.it)
  3. ensure that your console has a running config for
  4. awscli
  5. ssh (a public/private keys setup)
  6. kops
  7. kops create cluster --name k.devops.it --state=s3://kops-state-dev.devops.it --zones=eu-central-1a --node-count=2 --node-size=t2.micro --master-size=t2.micro --dns-zone=k.devops.it
  8. export KOPS_STATE_STORE=s3://kops-state-dev.devops.it
  9. kops update cluster k.devops.it --yes
  10. kops get cluster

When your exercise is done:

kops delete cluster --name k.devops.it --state=s3://kops-state-dev.cloud4wi.com

Test the cluster

kubectl run hello-minikube --image=k8s.gcr.io/echoserver:1.4 --port=8080
kubectl expose deployment hello-minikube --type=NodePort
kubectl get service
kubectl describe service hello-minikube
